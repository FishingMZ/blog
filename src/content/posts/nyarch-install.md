# 所用材料

笔记本（i7 Q820 + GTS 360m）、u盘一号（refus盘，16G）、u盘二号（老毛桃，32G）、refus、nyarch-linux.iso

# 前言

说实话，我是先知道nyarch才知道的arch，这听起来有点怪但确实是。

本来我是在b站刷到了评测，冲着这个系统的各种预装软件去vm装了虚拟机体验了一下，结果发现它的catgirl下载器和waifu下载器的图片都会缩得特别小，只有下载后才是正常的。

使用了一会之后行云流水地del了。😴

再后来我看到arch的自定义感觉特别帅特别好看，于是又去把nyarch的镜像翻出来，想装到笔记本上做个win+linux双系统来着。😋

# 过程

## 第一天

第一步我先去找了写盘的软件。

这个u盘我本来是做了ventory的，于是我直接把iso甩进去尝试进系统，结果笔记本连u盘都识别不到。😱

我在想是不是ventory有问题，遂换用了refus把nyarch（debian也试过）写进去了，结果还是读不到盘。

出师不利，我就想着等我上完学再回来研究。我在学校里感觉是因为笔记本识别不到引导，把硬盘拆下来到台式上面装会不会有说法呢？（大概我台式也读不到盘）😴

## 第二天

一回家我就看到有一个u盘放我笔记本上面了，是我爸的老毛桃u盘，神奇的是老毛桃一插上就可以检测到u盘并启动了。🤓

于是我马上把iso甩进老毛桃，再用老毛桃启动iso，果然进了nyarch的引导界面。

选择第一项，启动安装程序，然后我就卡在启动途中了，啥指令和reboot也不管用，报错提示好像是没读到某个文件。🧐

也对，这个u盘里面本质上还是老毛桃，我只是启动了iso而已，但是里面的文件没有存在于u盘里。我感觉又遇到瓶颈了。😱

## 正式开始

这个时候我突然灵光一现，随即用refus把我另外一个盘写进nyarch的iso文件，再把两个盘全插上笔记本。

首先，refus盘仍旧识别不到引导，我们用老毛桃u盘先启动里面的nyarch-iso，然后选择启动安装程序。

这个时候安装程序发现老毛桃u盘里面没有数据，就会读取另外一个refus盘的数据，上一次的报错立马消失了，两个盘实现了梦幻联动，无比顺利地进了live系统。

难道我真的是天才？🤓

# 问题

接下来就是在live系统里把系统装进硬盘，这就没啥好说的，点几下continue诗人都会。然后就是开机进bios把第二块硬盘的启动项改成第一位，顺利进入nyarch linux。😋

了吗？😱不出意外，还是出意外了，接下来1个多小时，nyarch一直在不断地花屏死机循环（不是完全花屏，就是花了一点点但是屏幕内容仍然可读那种）。

我想着是不是驱动问题，于是pacman也用了，yay也装了，就是整不上显卡驱动，罢了，下次还是用debian吧，可能arch还是不太适合我这种菜鸟（毕竟人家官网都说了）😵

# 附加

btw，我反而感觉我配置差很多的笔记本启动速度反而比我台式快一点呢？

两台我都部署了atlasos，不过我台式是win11笔记本是win10而已，而且我印象里win10atlas的启动也是没笔记本那么快的。

是硬盘的问题吗？这样看来我的笔记本硬盘得再拆一块下来给台式用了。
